## Decision

The target public API is GraphQL on NestJS. Do not rebuild the complete REST surface in NestJS as a prerequisite. Extract transport-neutral domain services, expose them through GraphQL, and retain the legacy Express routes until each consumer scenario has passed a semantic cutover gate. A temporary NestJS REST adapter is appropriate only when a specific client cannot move with its GraphQL operation.

Parity means the same authorization decisions, domain state transitions, side effects, and user-visible outcomes. It does not mean forcing GraphQL responses and errors to imitate REST envelopes byte-for-byte.

## Phase 1 GraphQL foundation now in place

The isolated `backend-v2/` workspace now boots NestJS 11 with code-first GraphQL and Apollo Server 5. It is not part of the legacy Railway build/start commands and receives no production traffic.

The first vertical foundation implements the existing access-cookie contract, verifies `itemize_auth` with the shared JWT secret, selects organization context only from `x-organization-id` or the user's database default, re-reads current membership and role from PostgreSQL, and propagates the verified identity through `AsyncLocalStorage`. A public `readiness` query and protected `viewerContext` query prove the middleware and guard chain.

The first domain slice exposes tenant-scoped `contacts` and `contact` queries plus single-row and bounded bulk contact mutations through a transport-neutral service and parameterized repository. Reads preserve legacy search, status, tag, assignee, sorting, and page semantics while adding strict inputs and deterministic ID tie-breaking. Writes serialize plan-limit checks, verify assignee membership, use organization-qualified locks and writes, and commit contact-created/contact-updated/tag workflow triggers and activities atomically with domain state. Updates preserve omitted fields and clear nullable fields only when null or empty is explicit. Bulk requests are capped at 100 IDs, deduplicated in request order, mutate only matched active-organization rows, and return requested/matched/changed/rejected sets without revealing whether a rejected ID is foreign or absent. Cross-tenant single-row reads and mutations return tenant-private `NOT_FOUND`. The comparison suite also found and repaired ambiguous unqualified columns in the legacy contact search query.

The canonical tag persistence checkpoint now gives each organization one normalized tag identity and stores contact/deal membership in junction rows. A one-time repair migration merges case drift, preserves the union of array and junction membership, rejects cross-tenant links, and installs bidirectional compatibility projection so retained array writers and future ID-based writers cannot silently diverge. Tag rename preserves the ID and updates contact/deal arrays; delete removes all membership; campaign and segment evaluators immediately see tags supplied through legacy arrays. Focused fresh-PostgreSQL coverage proves all of these cases, and the disposable runner now forwards optional Jest path arguments for fast clean-schema checkpoint reruns.

The canonical pipeline-stage checkpoint makes `pipeline_stages` authoritative while retaining `pipelines.stages` as a bidirectional compatibility projection. Historical repair trusts live JSON for overlapping definitions, removes unused stale shadow rows, preserves deal-referenced missing keys, and repairs empty pipelines. Composite deal foreign keys enforce both tenant ownership and stage membership, a partial unique index protects default-pipeline concurrency, and core deal/segment validation reads canonical rows. Focused fresh-PostgreSQL coverage proves JSON and direct-row projection, bounds and duplicate rejection, in-use deletion denial, direct tenant/stage constraint failures, default uniqueness, and idempotent drift repair.

The authenticated vocabulary checkpoint adds `TagsModule` and `PipelinesModule` to NestJS. Their ten operations read and mutate the canonical tag/stage stores with CSRF-protected writes, tenant-private misses, deterministic ordering and aggregates, case-insensitive tag uniqueness, projection-preserving rename/delete, stage validation/reordering, explicit nullable-description clearing, serialized default changes, in-use stage protection, and locked deletion checks. The pipeline board adapter preserves the current consumer shape and has independent read/mutation rollback flags. Fresh PostgreSQL runs 6 tag/pipeline parity cases in addition to the 24 contact/context cases.

The deal checkpoint adds `DealsModule` with deterministic filtered reads and all seven CSRF-protected writes. It preserves decimal precision as strings, validates pipeline/stage/contact/assignee tuples, serializes concurrent updates, and atomically records real stage/lifecycle transitions in the deal ledger, contact timeline when applicable, and durable workflow queue. Database checks prohibit contradictory terminal state. Two independent default-off frontend flags preserve REST rollback without coupling deal transport to pipeline-definition flags.

The authenticated forms checkpoint adds nine `FormsModule` operations for definition CRUD/duplication, atomic field replacement, and submission paging/deletion. The canonical contact-email checkpoint stores trimmed lowercase email or `NULL` at the database boundary while deliberately preserving duplicate and email-less contacts. ## Per-slice exit gate

A slice is ready to cut over only when all of the following are true:

- its ledger row has an explicit disposition and owner
- auth and tenant-denial cases pass
- success, validation, not-found, conflict, and failure paths are characterized
- database state and external side effects match the intended semantics
- retries are safe for mutations that can be repeated
- the frontend or external consumer uses the new operation successfully
- metrics/logs distinguish legacy and GraphQL execution
- rollback does not require data repair

The Express backend can be retired only after all externally used ledger rows are migrated, deliberately retained, or deliberately removed and the observation window shows no legacy traffic.
